# WooCommerce Markdown Guidelines

Use this skill when creating or editing markdown in WooCommerce.

## Critical Rules

1. Always lint after changes
2. Run markdown linting from the repository root
3. Use UTF-8 encoding for directory trees and special characters
4. Use ATX headings, fenced code blocks with language labels, and blank lines around lists and code blocks
