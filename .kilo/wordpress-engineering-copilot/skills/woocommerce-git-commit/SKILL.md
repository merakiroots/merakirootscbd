# Commit Changes

Commit uncommitted work with clear, conventional messages. Group unrelated changes into separate commits when appropriate.

## Core Rules

- Analyze changed files before proposing commit groups
- Default to a single commit unless changes are clearly unrelated
- Use concise verb-first imperative commit messages under 72 characters
- Preview proposed commits and wait for approval before executing
- Stage specific files only
- Never push to remote from this workflow
